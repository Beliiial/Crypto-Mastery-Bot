# Глобальный Промт: Разработка Telegram-бота "Gatee Bot" (Продажа доступа + CRM)

## 🎯 Цель проекта
Разработать профессионального Telegram-бота на `aiogram 3.x` для продажи доступа к приватному каналу. Бот должен включать: воронку продаж (бесплатный пробник), автоматизированную оплату через CryptoBot, реферальную систему, систему дожима (напоминания) и мощную админ-панель.

---

## 🛠️ Часть 1: Архитектура и База данных (Core)
**Задача:** Создать фундамент бота с масштабируемой структурой.
- **Стек:** Python 3.10+, aiogram 3.x, PostgreSQL (SQLAlchemy), Alembic (миграции).
- **База данных (Схемы):**
  - `User`: id, telegram_id, username, full_name, created_at, referral_id (кто пригласил), trial_received (bool), balance (float), sub_expires_at (datetime).
  - `Payment`: id, user_id, amount, currency, status (pending/success/failed), crypto_pay_id, created_at.
  - `Settings`: key, value (для хранения цен, ссылок на пробник и конфигов).
  - `Referral`: id, inviter_id, invitee_id, reward_amount.

---

## 🚀 Часть 2: Пользовательская воронка (User Flow)
**Задача:** Максимально конвертировать зашедшего пользователя в покупателя.
- **Хендлер `/start`:**
  - Глубокая ссылка (Deep Linking) для реферальной системы: `/start?ref=12345`.
  - Приветственный текст: *"Добро пожаловать в мир крипто-сигналов! Получи первый урок бесплатно."*
- **Модуль "Пробный курс":**
  - Кнопка "🎁 Получить пробник".
  - Бот выдает `file_id` (хранить в конфиге).
  - **Feature (CRM):** Если через 12/24/48 часов после получения пробника нет оплаты, бот шлет "дожимные" сообщения.
- **Модуль "Оплата" (CryptoPay API):**
  - Выбор тарифа (напр. 1 мес - 50 USDT, Навсегда - 150 USDT).
  - Создание инвойса в @CryptoBot через `aiocryptopay`.
  - Проверка оплаты через Webhook или Polling.
  - **Feature:** При успешной оплате бот генерирует `createChatInviteLink` с `member_limit=1`.

---

## 🤝 Часть 3: Виральность и Реферальная система
**Задача:** Снизить стоимость привлечения клиента.
- **Реферальный кабинет:**
  - Кнопка "🤝 Пригласи друга".
  - Выдача уникальной ссылки: `t.me/GateeBot?start=REF_ID`.
  - Статистика: сколько приглашено, сколько купило, заработанный баланс.
- **Механика вознаграждения:**
  - Если реферал покупает доступ, пригласивший получает +10% от суммы на внутренний баланс ИЛИ +7 дней к подписке (настраивается в админке).

---

## 👑 Часть 4: Админ-панель (God Mode)
**Задача:** Удобное управление бизнесом без прямого доступа к БД.
- **Главное меню админа:**
  - 📊 `Статистика`: Новые за сегодня, Всего оплат, Выручка (USDT), Конверсия в оплату.
  - ✉️ `Рассылка`: Создание сообщения (текст+фото+кнопки) и выбор аудитории (Все / Только оплатившие / Только те, кто взял пробник).
  - ⚙️ `Настройки`: Изменение цены, изменение ссылки на пробник, вкл/выкл тех. работ.
  - 👤 `Поиск юзера`: Поиск по ID/Username, просмотр его рефералов, ручное продление подписки.
  - 📋 `Логи`: Выгрузка последних 100 строк лога прямо в чат.

---

## 🛡️ Часть 5: Стабильность и Безопасность
- **Middleware:** 
  - Проверка подписки на основной канал (обязательное условие для использования бота).
  - Троттлинг (анти-спам).
- **Scheduler (APScheduler):**
  - Ежедневная проверка истекающих подписок.
  - Отправка уведомлений за 3 дня и за 1 день до конца доступа.
  - Автоматическое исключение из канала/чата (через `banChatMember` и `unbanChatMember`) при окончании подписки.
- **Logging:** Подробные логи всех финансовых операций и ошибок.

---

## 📝 Инструкция по выполнению (Step-by-Step)
1. **Подготовка:** Инициализировать репозиторий, настроить `.env` (BOT_TOKEN, CRYPTO_PAY_TOKEN, ADMIN_IDS).
2. **БД:** Написать модели SQLAlchemy и создать миграцию Alembic.
3. **Логика старта:** Реализовать /start с рефкой и выдачу пробника.
4. **Платежи:** Интегрировать CryptoPay API и логику выдачи инвайт-ссылок.
5. **CRM:** Настроить APScheduler для напоминаний и уведомлений об окончании подписки.
6. **Админка:** Реализовать меню статистики и рассылок.
7. **Тесты:** Проверить флоу оплаты и реферальную систему.
