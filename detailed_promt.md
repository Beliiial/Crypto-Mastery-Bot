# Техническое задание: Telegram-бот для продажи доступа в приватный канал (Gatee Bot)

## 1. Общее описание
**Цель:** Создать автоматизированную систему для продажи подписки на закрытый Telegram-канал/чат с использованием криптовалютных платежей (CryptoPay API) и встроенной системой удержания пользователей (CRM-функции).

---

## 2. Пользовательские функции (UX/UI)

### 2.1. Стартовый экран и Воронка (Onboarding)
- **Команда `/start`:**
  - Приветственное сообщение с кнопками: 🎁 `Получить пробный урок`, 💎 `Купить доступ`, ❓ `FAQ/Поддержка`.
  - Проверка: если пользователь уже в базе, не показывать кнопку "Получить пробный урок", если он его уже скачал.
- **Пробный урок (Trial):**
  - Бот отправляет файл (PDF/Видео) или ссылку.
  - **Feature:** Бот логирует факт получения пробника. Если через 24 часа пользователь не купил доступ, бот отправляет напоминание: *"Вам понравился урок? Успейте зайти в приватку со скидкой!"*.

### 2.2. Оплата (Payment Flow)
- **Интеграция с @CryptoBot (CryptoPay API):**
  - Генерация инвойса (USDT, TON, BTC).
  - Автоматическое подтверждение оплаты через Webhook или Polling.
  - Возможность выбора тарифов: `1 месяц`, `3 месяца`, `Навсегда`.
- **Ручная оплата (Backup):**
  - Если CryptoBot недоступен, админ может включить режим "Реквизиты". Пользователь отправляет скриншот, админ подтверждает вручную.

### 2.3. Доступ и Управление (Subscription)
- **Выдача доступа:**
  - После оплаты бот создает **одноразовую пригласительную ссылку** (`createChatInviteLink` с `member_limit=1`).
  - Бот уведомляет пользователя о завершении подписки за 3 дня, 1 день и в момент окончания (если подписка временная).
- **Личный кабинет:**
  - Статус подписки (активна/до какого числа).
  - Реферальная ссылка.

### 2.4. Реферальная система (Viral Growth)
- Каждый пользователь получает уникальную ссылку.
- **Бонус:** За каждого приглашенного, который купил доступ, пригласивший получает % от суммы на баланс в боте (который можно вывести или потратить на продление) ИЛИ +7 дней к своей подписке.

## 3. Административные функции (Admin Panel)

### 3.1. Управление пользователями
- Поиск пользователя по ID или @username.
- Просмотр истории платежей.
- Ручная выдача/отмена доступа.

### 3.2. Рассылки (Broadcasting)
- Создание рассылок с поддержкой Markdown, кнопок и медиафайлов.
- Таргетинг: "Всем", "Только тем, кто взял пробник, но не купил", "Только активным подписчикам".

### 3.3. Статистика и Аналитика
- **Dashboard:**
  - Кол-во новых пользователей за день/неделю.
  - Конверсия: `/start` -> `Пробник` -> `Оплата`.
  - Общий доход (в USDT).
  - Топ рефереров.

### 3.4. Динамическая настройка (Settings)
- Изменение цен на тарифы прямо в боте.
- Обновление файла/ссылки на пробный урок.
- Переключение режимов оплаты (CryptoBot / Manual).

## 4. Технический стек и Архитектура

### 4.1. Стек
- **Язык:** Python 3.10+
- **Фреймворк:** `aiogram 3.x` (асинхронность).
- **База данных:** `PostgreSQL` (через `SQLAlchemy` + `alembic` для миграций) — для надежности данных.
- **Очереди задач:** `APScheduler` (для напоминаний и проверки истечения подписки).
- **API Оплаты:** `aiocryptopay` (библиотека для CryptoPay).

### 4.2. Структура проекта
```text
gatee_bot/
├── src/
│   ├── handlers/          # Обработчики команд и кнопок
│   │   ├── admin/         # Админ-панель и рассылки
│   │   ├── users/         # Логика пользователя (старт, оплата, рефка)
│   │   └── common/        # Общие хендлеры
│   ├── middlewares/       # Проверка подписки, логирование, троттлинг
│   ├── services/          # Интеграция с CryptoBot, генерация ссылок
│   ├── database/          # Модели SQLAlchemy и CRUD операции
│   ├── utils/             # Напоминания, форматтеры, конфиг
│   └── keyboards/         # Все кнопки (Reply/Inline)
├── assets/                # Файлы для пробника, картинки
├── migrations/            # Alembic миграции
├── .env                   # Токены и настройки
├── main.py                # Точка входа
└── requirements.txt
```

## 5. Этапы разработки (Roadmap)

1. **Этап 1: База и старт.** Настройка проекта, БД, команда /start и выдача пробника.
2. **Этап 2: Оплата.** Интеграция с CryptoPay API и логика создания пригласительных ссылок.
3. **Этап 3: Админка.** Статистика, ручное управление пользователями и рассылки.
4. **Этап 4: CRM и Рефка.** Система напоминаний (авто-дожим) и реферальная программа.
5. **Этап 5: Полировка.** Обработка ошибок, логирование, деплой на VPS (Docker).

---

## 6. Безопасность и Стабильность
- **Троттлинг:** Защита от спама кнопками.
- **Логирование:** Использование `structlog` или стандартного `logging` для отслеживания ошибок.
- **Валидация:** Проверка входящих данных от CryptoBot через хеш (защита от поддельных уведомлений).
- **Backup:** Ежедневный бэкап базы данных.
