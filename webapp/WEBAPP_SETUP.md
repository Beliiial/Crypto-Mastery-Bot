# Подключение WebApp к Telegram-боту

## 1. Подготовка приложения (WebApp)

Перед интеграцией в бот, ваше приложение должно быть доступно по HTTPS URL.

### Локальная разработка (через ngrok)
Если вы запускаете приложение локально (`npm run dev`), используйте [ngrok](https://ngrok.com/) для создания публичного туннеля:
```bash
ngrok http 3000
```
Скопируйте полученный HTTPS адрес (например, `https://a1b2c3d4.ngrok.io`).

### Продакшн (Vercel)
1. Загрузите код на GitHub.
2. Подключите репозиторий в Vercel.
3. Получите домен (например, `https://your-project.vercel.app`).

## 2. Настройка бота через BotFather

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. Выберите вашего бота через команду `/mybots`.
3. Перейдите в **Bot Settings** -> **Menu Button**.
4. Нажмите **Configure Menu Button**.
5. Отправьте ссылку на ваше приложение (из шага 1).
6. Введите текст кнопки, например "Открыть приложение".

Теперь у пользователей появится кнопка слева от поля ввода сообщения, открывающая ваш WebApp.

## 3. Интеграция данных пользователя (initData)

Ваше приложение автоматически пытается получить данные о пользователе из Telegram.
В файле `lib/user-context.tsx` реализована базовая логика. Для полноценной проверки подлинности данных (валидация хеша) на бэкенде, вам потребуется:

1. Получить `initData` из `window.Telegram.WebApp.initData`.
2. Отправить эту строку на ваш сервер.
3. Проверить подпись, используя токен вашего бота.

## 4. Админ-панель

В текущей версии реализована скрытая админ-панель.
Чтобы получить доступ:
1. Зайдите в Профиль.
2. Нажмите на скрытую кнопку (иконка шестеренки) в правом верхнем углу.
3. Вы получите права администратора и увидите кнопку "Админ-панель".

Через админ-панель вы можете:
- Добавлять и удалять курсы.
- Добавлять менторов.
- Управлять пользователями.

## 5. Логика Чатов и Курсов

- **Курсы**: Изначально список пуст. Вы должны добавить их через админ-панель.
- **Чаты**: Пользователь видит 3 чата (Gatee, Agwwee, Support).
  - **Support**: Доступен всем.
  - **Gatee/Agwwee**: Доступны только после покупки "Личного менторства".
- **Оплата**:
  - Нажатие "Оплатить" в модальном окне переводит заявку в статус ожидания.
  - Пользователь видит кошелек для оплаты.
  - После нажатия "Я оплатил", доступ выдается (в демо-режиме - с задержкой, в реальном - после проверки админом или API).
